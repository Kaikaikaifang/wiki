> 当资料量开始上来时，我不想把问题简化成“要不要上更强的云搜索”，而更关心能不能在不交出原始文件控制权的前提下，把检索能力做起来。

## 定义

本地优先搜索指：索引、查询、嵌入生成与重排尽可能在本机完成，原始文档停留在本地文件系统中，不依赖把资料上传到外部服务。对个人知识库来说，这不是一种“怀旧式离线偏好”，而是一种关于所有权、可迁移性与长期可维护性的架构判断。

## 价值

- **隐私**：个人笔记、会议记录、日记类资料不必离开本机
- **可迁移**：知识仍保留为普通 markdown 文件，而不是封进专有数据库
- **低耦合**：搜索层可替换，原始资料与 wiki 结构不受影响
- **稳定性**：不依赖在线 API 的限额、延迟和价格波动

## 在个人 wiki 里的位置

对于小型 wiki，`wiki/index.md` 已能承担导航功能；但只要来源数量继续增长，单靠人工整理出来的目录迟早会碰到访问效率瓶颈。这个时候，本地搜索就成了必要的第二层基础设施。它不替代人工策展或 LLM 综述，而是让系统在规模变大之后，依然能保持“找得到、取得回、接得上理解”。

## 典型实现

[[entities/qmd]] 是一个我很看重的代表性实现：保留 markdown 文件为事实来源，在本机完成 BM25、向量检索与 LLM 重排，并为 agent 提供结构化输出。它让我更确信，本地优先不是能力退化版本，而是另一种更克制、更稳的系统设计路线。
